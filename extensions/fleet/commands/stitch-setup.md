# Stitch Implement — Setup

Interactive setup command that discovers your Stitch projects and design master screens, then writes a config override so `stitch-prototype` and `stitch-validate` reuse them.

## Prerequisites

- The **Stitch MCP server** must be running and connected (`stitch/list_projects` available).
- Install the companion skills for the full Stitch workflow:

```sh
npx skills add https://github.com/google-labs-code/stitch-skills --skill stitch-loop
npx skills add https://github.com/google-labs-code/stitch-skills --skill taste-design
```

## Concepts

### Project IDs

Every Stitch project has a numeric ID (e.g. `6139132077804554844`). The resource name follows the pattern `projects/{id}`. Pinning a project ID in your config prevents the extension from creating throwaway projects on every run — it reuses the existing one.

### Design Master IDs

Each screen inside a Stitch project has a hex ID (e.g. `d7237c7d78f44befa4f60afb17c818c1`). A **design master** is a screen you designate as the visual reference. The prototype command loads the master's design theme, colours, typography, and layout patterns and applies them when generating new screens — ensuring visual consistency.

---

## Step 1 — Verify Stitch MCP connection

Call `stitch/list_projects` with no arguments. If the call fails, inform the user:

> Stitch MCP server is not reachable. Start it and try again.

If it succeeds, proceed.

## Step 2 — List projects

<!-- GREP:STITCH-SETUP-LIST-PROJECTS -->

Display the returned projects in a numbered list:

```
#  | Project ID           | Title           | Device   | Screens
---|----------------------|-----------------|----------|--------
1  | 6139132077804554844  | My App          | MOBILE   | 5
2  | 7821445009123456789  | Marketing Site  | DESKTOP  | 12
```

Use `vscode_askQuestions` to let the user select one or more projects:

> Which projects should the stitch-implement extension use? (comma-separated numbers, or "all")

## Step 3 — Inspect selected projects

For each selected project, call `stitch/get_project` to retrieve the full metadata including `designTheme`, `deviceType`, and `projectType`. Record these.

## Step 4 — List screens for each project

<!-- GREP:STITCH-SETUP-LIST-SCREENS -->

For each selected project, call `stitch/list_screens` and display:

```
#  | Screen ID                        | Name        | Size
---|----------------------------------|-------------|----------
1  | d7237c7d78f44befa4f60afb17c818c1 | index       | 390×1249
2  | a1b2c3d4e5f6789012345678abcdef01 | dashboard   | 1440×900
```

Use `vscode_askQuestions`:

> Which screens are design masters for "{project_title}"? (comma-separated numbers, or "none")

Design masters are the screens whose visual style you want replicated across new screens. Pick your most polished, representative screens.

## Step 5 — Write config override

<!-- GREP:STITCH-SETUP-CONFIG-WRITE -->

Build a YAML config override and write it to:

```
.specify/extensions/stitch-implement/stitch-implement-config.local.yml
```

The file structure:

```yaml
# Auto-generated by speckit.fleet.stitch-setup
# Re-run the setup command to refresh.
stitch:
  projects:
    - id: "{project_id}"
      name: "{project_title}"
      device_type: "{DESKTOP|MOBILE|TABLET}"

  design_masters:
    - id: "{screen_id}"
      project_id: "{project_id}"
      label: "{screen_name}"
```

Create the directory path if it does not exist.

## Step 6 — Recommend companion skills

Print:

```
✅ Config written. Your stitch-implement commands will now reuse these projects and design masters.

Recommended companion skills (install once):

  npx skills add https://github.com/google-labs-code/stitch-skills --skill stitch-loop
  npx skills add https://github.com/google-labs-code/stitch-skills --skill taste-design

• stitch-loop — autonomous baton-passing loop for iterative website building
• taste-design — generates DESIGN.md with premium design system (atmosphere, colours, typography, motion)
```

## Step 7 — Summary

Show the final config that was written, and confirm the number of projects and design masters saved.
